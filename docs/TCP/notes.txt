🌐 3. TCP/IP
1. Netværkslag (TCP/IP og OSI)
🧱 OSI-modellen (7 lag)
Physical (kabler, signaler)
Data Link (MAC, switches)
Network (IP, routing)
Transport (TCP/UDP)
Session
Presentation
Application (HTTP, DNS osv.)
🌐 TCP/IP modellen (4 lag)
Link (Data Link + Physical)
Internet (IP)
Transport (TCP/UDP)
Application (HTTP, DNS osv.)

👉 TCP/IP er den praktiske model, OSI er den teoretiske.

2. IP-adresse vs MAC-adresse
🌍 IP-adresse
Logisk adresse
Kan ændres
Bruges til routing over netværk
🔗 MAC-adresse
Hardware adresse (NIC)
Unik for netværkskort
Bruges lokalt i netværk
3. Hvorfor kan jeg ikke se MAC på ek.dk?

Fordi:

MAC adresser virker kun på lokalnet (LAN)
Routere fjerner Layer 2 information
Internettet arbejder på IP (Layer 3)

👉 Wireshark kan kun se MAC på dit eget netværk

4. Router vs Switch
🔀 Switch
Arbejder på MAC niveau
Sender data internt i LAN
🌐 Router
Arbejder på IP niveau
Forbinder netværk (LAN → Internet)
📡 PRAKTISKE SPØRGSMÅL
5. Lokal IP vs public IP
Lokal IP
Fx 192.168.x.x
Bruges internt i netværk
Public IP
Synlig på internettet
Tildeles af ISP

7. Hvad er en port?
Identificerer service på en maskine
IP = maskine
Port = program
Eksempler:
80 → HTTP
443 → HTTPS
22 → SSH
3306 → MySQL
8. src og dst IP
src = sender
dst = modtager

👉 i en pakke:

src IP = hvor den kommer fra
dst IP = hvor den går hen
9. src port og dst port
src port = midlertidig klientport
dst port = service port

Eksempel:

src: 51234 → dst: 443

👉 klient → webserver

10. SYN, ACK, FIN (TCP flags)
🔗 SYN
Start connection
✅ ACK
Bekræftelse
🔚 FIN
Luk connection
TCP handshake:
SYN
SYN-ACK
ACK

👉 ses i Wireshark under “Info” kolonnen

11. TCP vs UDP
TCP
Forbindelsesorienteret
Pålidelig
Har ACK, retransmission
UDP
Hurtig
Ingen garanti
Bruges til streaming, gaming
12. Sequence + ACK numbers
Sequence number
Rækkefølge af data
ACK number
Bekræfter modtaget data

👉 bruges til:

korrekt levering
flow control
